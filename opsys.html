Operating Systems Terminology

TERM DEFINITION
Operating System The Program in a computer that can read and execute a computer's basic functions.
Job Queue A storage organization in which jobs are ordered in first-in, first-out.
FIFO The abreviation of First-in, First-out, where the objects are removed from queue in the order in which they arrived.
Interactive processing A development that let operating systems have a program being executed to cary on a dialogue with the user.
Time-sharing A technique where time is divided into intervals and the execution of each job is restricted to one interval at a time.
Multiprogramming A feature wehre operating systems can provide service to multiple users at the same time.
Application software A program made to carry out tasks other than one that makes the operation of the computer.
System software Software that performs the tasks that are common to computer systems in general.
Utility software Consists of software units that extend the capabilities of the operating systems
Graphical user interface (GUI) An interface in which a user can interact with electronic devices.
Shell User interfaces that communicated with users through textual messages using a keyboard and monitor screen.
Kernel An internal part of the operating system that contains software components that perfrom very basic functions required by the computer installation.
Device driver Software units that communicate with the controllers to carry out operations on the peripheral devices attached to the machine.
File manager A unit that coordinates the use of the machine's mass storage facilities.
Memory manager A component of the kernel that coordinates the machine's use of the main memory.
Virtual memory A technique of creating "fictional" space.
Boot strapping The procedure that transfers the operating system from mass storage into main memory.
ROM Read- only memory that can only be read not altered.
Deadlock The condition in which two or more processes are blocked from progressing because each is waiting for a resource that is allocated to another.
Spooling A technique of holding data for output at a later but more convenient time.
Super user (or administrator) Someone who can kill or remove some of processes happening.
Multi-factor authentication The technique in which to use more than one piece of evidence to gain access to accounts etc.
